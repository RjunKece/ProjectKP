BAB IV
HASIL DAN PEMBAHASAN

4.1 Hasil Implementasi Sistem

Pada bab ini diuraikan hasil implementasi Sistem ERP Prototype untuk PT. Golden Intan Berlian yang telah dibangun menggunakan framework Laravel 12 dengan bahasa pemrograman PHP 8.2. Sistem ini dirancang sebagai aplikasi web berbasis arsitektur Model-View-Controller (MVC) yang berjalan di lingkungan server lokal menggunakan Laragon. Basis data yang digunakan adalah MySQL dengan nama database erp-prototype. Frontend dikembangkan menggunakan Blade template engine dengan dukungan TailwindCSS 4.0 dan Alpine.js untuk interaktivitas sisi klien, serta Vite 7 sebagai build tool untuk asset bundling.

Sistem ERP ini memiliki dua peran utama pengguna, yaitu Super Admin dan Karyawan. Super Admin memiliki akses penuh terhadap seluruh fitur sistem termasuk manajemen pengguna, monitoring aktivitas, dan pembuatan laporan. Sementara itu, Karyawan memiliki akses terbatas pada dashboard pribadi dan profil masing-masing.

4.2 Implementasi Basis Data

Struktur basis data sistem ERP dibangun menggunakan fitur migration pada Laravel. Terdapat lima tabel utama yang menjadi fondasi sistem, yaitu tabel users, roles, divisions, activities, dan reports. Selain itu terdapat tabel pendukung bawaan Laravel seperti sessions, password_reset_tokens, cache, dan jobs.

4.2.6 Relasi Antar Tabel

Relasi antar tabel dalam sistem ERP ini menggunakan pola Eloquent Relationship pada Laravel. Berikut adalah hubungan antar entitas:

  • User belongsTo Role — Setiap pengguna memiliki satu role.
  • User belongsTo Division — Setiap pengguna dapat memiliki satu divisi (nullable).
  • User hasMany Activity — Satu pengguna dapat memiliki banyak aktivitas.
  • Activity belongsTo User — Setiap aktivitas dimiliki oleh satu pengguna.
  • Report belongsTo User (creator) — Setiap laporan dibuat oleh satu pengguna.
  • Role hasMany User — Satu role dapat dimiliki oleh banyak pengguna.
  • Division hasMany User — Satu divisi dapat memiliki banyak pengguna.

4.3 Implementasi Antarmuka Pengguna

4.3.1 Halaman Login

Halaman login merupakan pintu masuk utama sistem ERP. Desain halaman login menggunakan layout dua kolom (split screen) dengan bagian kiri berisi formulir login dan bagian kanan menampilkan branding perusahaan PT. Golden Intan Berlian. Halaman ini menggunakan skema warna emas (gold) sebagai identitas visual perusahaan dengan gradien dari #f5e6b3 ke #d4af37.

Formulir login memvalidasi dua field yaitu email dan password. Proses autentikasi ditangani oleh LoginController yang menggunakan facade Auth::attempt() dari Laravel. Jika kredensial valid, pengguna akan diarahkan ke dashboard sesuai role masing-masing. Jika gagal, sistem menampilkan pesan error "Email atau password salah."

4.3.2 Dashboard Admin

Dashboard admin merupakan halaman utama setelah Super Admin berhasil login. Halaman ini menampilkan ringkasan performa sistem yang terbagi menjadi tiga bagian utama:

  1. System Snapshot — Menampilkan enam KPI (Key Performance Indicator) berupa Total Users, Active Employees, Total Activities, Today Activity, Productivity (estimasi bulanan), dan Growth (persentase pertumbuhan).
  2. Activity Analytics — Menampilkan grafik line chart menggunakan library Chart.js yang memvisualisasikan tren aktivitas karyawan dan super admin selama 12 bulan terakhir.
  3. Quick Actions — Menyediakan pintasan navigasi cepat ke fitur Manage Users, Review Activities, dan Generate Reports.

4.3.3 Halaman User Management

Halaman User Management menyediakan antarmuka untuk mengelola akun pengguna dalam sistem. Fitur-fitur yang tersedia meliputi:

  1. Summary Cards — Menampilkan statistik Total Users, Active Users, Super Admin, dan Pending Approval.
  2. Filter dan Pencarian — Toolbar untuk mencari pengguna berdasarkan nama atau email, serta filter berdasarkan role dan divisi.
  3. Tabel Data Pengguna — Menampilkan daftar pengguna dengan informasi nama, email, role, divisi, status, dan tombol aksi.
  4. Tambah User — Modal form untuk menambahkan pengguna baru dengan field nama, email, role, dan divisi. Password default ditetapkan sebagai "password123" yang di-hash menggunakan bcrypt.
  5. Reset Password — Fitur untuk mereset password pengguna kembali ke default melalui modal konfirmasi.

4.3.4 Halaman Monitoring Aktivitas

Halaman Monitoring Aktivitas menampilkan seluruh log aktivitas yang terjadi dalam sistem. Halaman ini memiliki komponen:

  1. KPI Summary — Empat kartu indikator menampilkan Total Aktivitas, Aktivitas Hari Ini, Aktivitas Admin, dan Aktivitas Karyawan.
  2. Filter Bar — Toolbar pencarian dengan filter berdasarkan role, divisi, dan tanggal.
  3. Activity Table — Tabel yang menampilkan informasi user, role, divisi, deskripsi aktivitas, dan waktu. Data ditampilkan dengan paginasi 15 item per halaman.

4.3.5 Halaman Reports Center

Halaman Reports Center merupakan pusat analisis dan pembuatan laporan strategis. Halaman ini terdiri dari:

  1. KPI Strip — Lima indikator menampilkan Total Reports, Generated (Month), Scheduled, System Coverage, dan Storage Used.
  2. Report Builder — Empat modul pembuatan laporan yaitu Financial Intelligence, Department Performance, Growth & Trends, dan Custom Intelligence. Masing-masing modul membuka modal form untuk mengisi detail laporan.
  3. Recent Reports — Tabel laporan terbaru dengan informasi judul, tipe, periode, status, dan aksi (View/Download). Data dipaginasi 10 item per halaman.
  4. System Insight — Panel informasi menampilkan statistik penggunaan laporan seperti Most Used Report, Least Active Module, Last Generated, dan Automation Status.

4.3.6 Halaman Profil Pengguna

Halaman profil menampilkan informasi akun pengguna yang sedang login. Terdapat tiga section utama:

  1. Profile Overview — Avatar inisial, nama, email, badge role, dan status aktif.
  2. Account Information — Detail informasi role, divisi, dan tanggal registrasi.
  3. Activity Summary — Ringkasan total aktivitas dan aktivitas bulan berjalan.

4.4 Implementasi Backend

4.4.1 Arsitektur MVC

Sistem ERP ini mengimplementasikan pola arsitektur Model-View-Controller (MVC) yang merupakan standar pada framework Laravel. Struktur kode dibagi menjadi tiga lapisan utama:

  1. Model — Terdapat lima model utama yaitu User, Role, Division, Activity, dan Report. Setiap model mendefinisikan fillable attributes, casting, dan relasi Eloquent.
  2. View — Menggunakan Blade template engine dengan sistem layout inheritance. Layout utama admin (layouts/admin.blade.php) menjadi template induk untuk seluruh halaman panel admin.
  3. Controller — Terdapat delapan controller yang menangani logika bisnis, termasuk AdminDashboardController, UserController, ActivityController, ReportController, LoginController, dan ProfileController.

4.4.2 Sistem Autentikasi

Sistem autentikasi menggunakan Laravel Breeze sebagai scaffolding. Proses login diimplementasikan melalui LoginController dengan alur: validasi input email dan password, autentikasi menggunakan Auth::attempt(), regenerasi session untuk keamanan, dan redirect ke dashboard berdasarkan role pengguna. Sistem membedakan redirect antara Super Admin (ke admin/dashboard) dan Karyawan (ke karyawan/dashboard) melalui middleware auth pada route /dashboard.

4.4.3 Sistem Routing

Routing sistem dikonfigurasi dalam file routes/web.php dengan pengelompokan route berdasarkan middleware dan prefix. Terdapat empat kelompok route utama:

  1. Auth Routes — Route untuk login (GET dan POST /login).
  2. Admin Routes — Route dengan prefix /admin dan middleware auth, mencakup dashboard, users, activities, dan reports.
  3. Karyawan Routes — Route dengan prefix /karyawan dan middleware auth untuk dashboard karyawan.
  4. Utility Routes — Route untuk profile dan logout.

4.4.4 Service Layer

Sistem mengimplementasikan service layer melalui kelas ActivityLogger yang berfungsi sebagai helper untuk mencatat aktivitas pengguna secara konsisten. Service ini menyediakan method statis log() yang menerima parameter deskripsi dan status, kemudian membuat record baru pada tabel activities dengan informasi user_id, tanggal, deskripsi, dan status.

4.4.5 Fitur Dark Mode

Sistem mendukung fitur dark mode yang diimplementasikan menggunakan Alpine.js. Tema disimpan di localStorage browser dengan key 'erp_theme'. Fungsi themeManager() mengelola inisialisasi tema, toggle antara light dan dark mode, serta penerapan class 'dark' pada elemen HTML root. TailwindCSS dikonfigurasi dengan darkMode: 'class' untuk mendukung styling kondisional berdasarkan tema aktif.

4.6 Pembahasan

4.6.1 Kelebihan Sistem

Berdasarkan hasil implementasi dan pengujian, sistem ERP Prototype PT. Golden Intan Berlian memiliki beberapa kelebihan sebagai berikut:

  1. Arsitektur Terstruktur — Penggunaan pola MVC pada Laravel memastikan pemisahan antara logika bisnis, tampilan, dan data sehingga kode lebih mudah dipelihara dan dikembangkan.
  2. Keamanan — Sistem menerapkan hashing password menggunakan bcrypt, CSRF protection pada setiap form, session regeneration setelah login, dan validasi input pada sisi server.
  3. Responsif dan Modern — Antarmuka pengguna dibangun dengan TailwindCSS yang memastikan tampilan responsif di berbagai ukuran layar. Fitur dark mode meningkatkan kenyamanan pengguna.
  4. Logging Aktivitas — Setiap aksi penting dalam sistem dicatat secara otomatis melalui ActivityLogger service, memungkinkan audit trail yang komprehensif.
  5. Modular — Struktur kode yang modular memudahkan penambahan fitur baru tanpa mengganggu fitur yang sudah ada.

4.6.2 Keterbatasan Sistem

Meskipun demikian, sistem ini juga memiliki beberapa keterbatasan yang perlu diperhatikan:

  1. Prototipe — Sistem masih berupa prototipe sehingga beberapa fitur seperti dashboard karyawan belum diimplementasikan secara lengkap.
  2. Middleware Role — Belum ada middleware khusus untuk memvalidasi role pengguna pada setiap route admin, sehingga karyawan yang mengetahui URL admin secara teoritis dapat mengaksesnya.
  3. File Upload — Fitur upload lampiran pada report masih menggunakan penyimpanan lokal dan belum terintegrasi dengan cloud storage.
  4. Notifikasi — Belum ada sistem notifikasi real-time untuk memberitahu admin tentang aktivitas penting.
  5. Export Data — Fitur download report belum diimplementasikan sepenuhnya untuk menghasilkan file PDF atau Excel.
