4.x.10 Activity Diagram — Melihat Dashboard Karyawan

Nama Use CaseMelihat Ringkasan Performa Individual
AktorKaryawan
DeskripsiKaryawan melihat halaman dashboard personal yang menampilkan ringkasan performa individu, termasuk statistik aktivitas, grafik tren, aktivitas terbaru, dan akses cepat ke fitur-fitur utama. Data yang ditampilkan hanya milik karyawan yang bersangkutan (difilter berdasarkan auth()->user()->id).
Pre-conditionKaryawan telah berhasil login. Middleware memverifikasi role = "karyawan".
Post-conditionDashboard karyawan ditampilkan dengan data personal dari database.

Alur Utama (Main Flow):

  1. Karyawan mengklik menu "Dashboard" pada sidebar navigasi atau otomatis diarahkan setelah login.
  2. Sistem menerima request GET ke route /karyawan/dashboard dan menjalankan KaryawanDashboardController@index.
  3. Sistem mengirimkan query ke Database dengan filter WHERE user_id = auth()->user()->id:
    • Query total aktivitas personal.
    • Query aktivitas bulan ini: WHERE MONTH(tanggal) = MONTH(CURDATE()).
    • Query aktivitas minggu ini.
    • Query aktivitas hari ini.
    • Query tren 6 bulan terakhir untuk chart.
    • Query 5 aktivitas terbaru: ORDER BY created_at DESC LIMIT 5.
  4. Database mengeksekusi seluruh query dan mengembalikan data personal ke Sistem.
  5. Sistem memproses data dan merender halaman dashboard (karyawan.dashboard) yang menampilkan:
    • Welcome Banner: Salam personal dengan nama karyawan, badge role, dan divisi.
    • 4 KPI Cards (Personal Snapshot): Total Aktivitas, Bulan Ini, Minggu Ini, dan Hari Ini.
    • Tren Aktivitas Chart: Bar chart aktivitas 6 bulan terakhir menggunakan Chart.js.
    • Aktivitas Terbaru: Tabel 5 log aktivitas terkini (tanggal, deskripsi, status).
    • Quick Actions: Tombol akses cepat ke Aktivitas Saya, Laporan Perusahaan, dan Overview.
  6. Karyawan melihat ringkasan performa personal dan dapat bernavigasi ke fitur lain.

4.x.11 Activity Diagram — Melihat dan Mencatat Aktivitas Individual

Nama Use CaseMelihat Aktivitas Individual dan Menambah Aktivitas
AktorKaryawan
DeskripsiKaryawan melihat riwayat aktivitas kerja harian miliknya sendiri, melihat progres target harian divisi, dan dapat mencatat aktivitas baru dengan melampirkan file atau link referensi. Fitur ini terhubung langsung dengan modul monitoring admin.
Pre-conditionKaryawan telah login. Daily targets untuk divisi karyawan telah dikonfigurasi.
Post-conditionAktivitas baru berhasil tersimpan di database dan terlihat di halaman monitoring admin.

Alur Utama (Main Flow) — Mencatat Aktivitas Baru:

  1. Karyawan mengklik menu "Aktivitas Saya" pada sidebar navigasi.
  2. Sistem menerima request GET ke route /karyawan/activities dan menjalankan KaryawanActivityController@index.
  3. Sistem mengirimkan query ke Database:
    • Query aktivitas personal: SELECT * FROM activities WHERE user_id = ? ORDER BY tanggal DESC.
    • Query daily targets: SELECT * FROM daily_targets WHERE division_id = ? AND is_active = true.
    • Query KPI: total, hari ini, bulan ini, minggu ini (semua difilter user_id).
  4. Database mengembalikan data dan Sistem merender halaman yang menampilkan:
    • 4 KPI Cards: Total Aktivitas, Hari Ini, Bulan Ini, Minggu Ini.
    • Target Harian: Progress bar per target divisi dengan persentase pencapaian.
    • Chart Tren 7 Hari: Bar chart aktivitas mingguan.
    • Filter Bar: Pencarian, filter status, dan filter tanggal.
    • Tabel Aktivitas: Kolom nomor, tanggal, deskripsi, status, dan lampiran dengan pagination.
  5. Karyawan melihat riwayat aktivitas dan progres target harian.
  6. Karyawan mengklik tombol "➕ Tambah Aktivitas".
  7. Sistem menampilkan modal form "Tambah Aktivitas Baru" yang berisi field:
    • Tanggal: Date picker (default: hari ini, wajib).
    • Target Harian: Dropdown daftar target divisi (opsional).
    • Deskripsi Aktivitas: Textarea untuk deskripsi detail (wajib).
    • Lampiran File: Upload file dokumen (opsional, max 10MB).
    • Link Referensi: Input URL referensi eksternal (opsional).
  8. Karyawan mengisi field yang diperlukan.
  9. Karyawan mengklik tombol "Simpan".
  10. Sistem menerima request POST ke /karyawan/activities dan menjalankan KaryawanActivityController@store.
  11. Sistem melakukan validasi (tanggal wajib, deskripsi wajib, file max 10MB jika ada).
  12. Jika terdapat file lampiran, Sistem mengupload dan menyimpan file ke storage.
  13. Sistem mengirimkan query INSERT ke Database: INSERT INTO activities (user_id, tanggal, deskripsi, status, file_path, file_name, link, target_id) VALUES (...).
  14. Database menyimpan record dengan status default "submitted".
  15. Sistem menutup modal, menampilkan notifikasi sukses, dan merefresh halaman.
  16. Karyawan melihat aktivitas baru muncul pada tabel dan progres target harian diperbarui.

Alur Alternatif — Filter Riwayat:

  1. Karyawan Memfilter Aktivitas:
    1. Karyawan mengisi parameter filter (status atau tanggal).
    2. Karyawan mengklik tombol "Filter".
    3. Sistem mengirimkan query dengan klausa WHERE tambahan ke Database.
    4. Sistem merender ulang tabel dengan hasil filter.
  2. Karyawan Mengubah Status Aktivitas:
    1. Karyawan mengklik tombol edit pada baris aktivitas.
    2. Sistem menampilkan modal edit dengan field status (dropdown) dan lampiran.
    3. Karyawan mengubah status (misalnya dari "submitted" ke "completed").
    4. Karyawan mengklik "Update".
    5. Sistem mengirimkan request PUT ke /karyawan/activities/{id}.
    6. Database memperbarui record aktivitas.

4.x.12 Activity Diagram — Melihat Daftar Laporan Perusahaan

Nama Use CaseMelihat Daftar Laporan Perusahaan yang Telah Dipublikasikan
AktorKaryawan
DeskripsiKaryawan mengakses Pusat Laporan untuk melihat laporan perusahaan yang telah dipublikasikan oleh admin, laporan divisi terkait, dan laporan yang dibuat sendiri. Karyawan juga dapat membuat laporan baru dan memberikan tanggapan (respon) terhadap laporan yang ada.
Pre-conditionKaryawan telah login. Terdapat minimal satu laporan yang telah dipublikasikan.
Post-conditionKaryawan berhasil melihat daftar laporan dan/atau memberikan respon.

Alur Utama (Main Flow) — Melihat Laporan Perusahaan:

  1. Karyawan mengklik menu "Laporan" pada sidebar navigasi.
  2. Sistem menerima request GET ke route /karyawan/reports dan menjalankan KaryawanReportController@index.
  3. Sistem mengirimkan query ke Database:
    • Laporan perusahaan: SELECT * FROM reports WHERE scope = 'company' ORDER BY created_at DESC.
    • Laporan divisi: SELECT * FROM reports WHERE scope = 'division' AND division_id = ?.
    • Laporan sendiri: SELECT * FROM reports WHERE created_by = ?.
    • KPI: jumlah per kategori dan jumlah belum dibaca.
  4. Database mengembalikan data dan Sistem merender halaman Pusat Laporan (karyawan.reports.index):
    • 4 KPI Cards: Laporan Perusahaan, Divisi Saya, Laporan Saya, dan Belum Dibaca.
    • Tab Navigasi: Tiga tab — "🏛️ Perusahaan", "🏢 Divisi Saya", dan "📄 Laporan Saya".
    • Daftar Laporan: List item dengan judul, deskripsi singkat, tanggal, pembuat, dan badge prioritas.
  5. Karyawan melihat daftar laporan pada tab "Perusahaan" (default).
  6. Karyawan mengklik salah satu judul laporan untuk melihat detail.
  7. Sistem mengirimkan request GET ke /karyawan/reports/{id}.
  8. Database mengambil detail laporan beserta daftar tanggapan (report_responses).
  9. Sistem menampilkan modal detail laporan yang berisi: header, isi lengkap, metadata, dan daftar respon dari pengguna lain.
  10. Karyawan membaca isi laporan lengkap.

Alur Alternatif — Memberikan Tanggapan:

  1. Karyawan Mengirim Respon:
    1. Pada modal detail laporan, karyawan mengisi field "Tulis tanggapan..." pada bagian bawah.
    2. Karyawan mengklik tombol "Kirim".
    3. Sistem menerima request POST ke /karyawan/reports/{id}/reply dan menjalankan KaryawanReportController@reply.
    4. Sistem memvalidasi pesan (tidak boleh kosong).
    5. Sistem mengirimkan query INSERT ke Database: INSERT INTO report_responses (report_id, user_id, message, type) VALUES (?, ?, ?, 'reply').
    6. Database menyimpan respon dan Sistem merefresh daftar tanggapan pada modal.
    7. Karyawan melihat respon baru muncul pada daftar tanggapan.
  2. Karyawan Membuat Laporan Baru:
    1. Karyawan mengklik tombol "✍️ Buat Laporan Baru".
    2. Sistem menampilkan modal form dengan field: Judul, Tipe, Scope, dan Isi Laporan.
    3. Karyawan mengisi field dan mengklik "Kirim Laporan".
    4. Sistem menerima request POST ke /karyawan/reports.
    5. Database menyimpan laporan baru dengan created_by = auth()->user()->id.
    6. Sistem menampilkan notifikasi sukses dan merefresh daftar laporan.
  3. Karyawan Berpindah Tab:
    1. Karyawan mengklik tab "🏢 Divisi Saya" atau "📄 Laporan Saya".
    2. Sistem memfilter data laporan berdasarkan tab yang aktif.
    3. Daftar laporan diperbarui sesuai kategori yang dipilih.
